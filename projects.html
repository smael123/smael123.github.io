Personality Patterns among US Politicians

This was my Senior Project. I wanted to find personality patterns among US politicians (hence the title) using Twitter data. I was to gather the tweets, get personality percentile scores for each politician based on their concatenated tweets.

Gathering Tweets

First I got an excel file with the names of all the people in the House of Representatives from their website

I used the python-twitter API and the Python programming language. Using this API I first gathered the names of the people in the House of Representatives. It was from a twitter list by Twitter Government but a I write this the link I used is dead. I used the GET lists/list part of the API

Using openpyxl, I inserted the twitter handle (screen name) to the corresponding name based on whether or not the real name matched. It almost worked fully. Some had middle names in the government file that did not match the Twitter name. And some used suffixes like Jr. that were again not in the Twitter name. For those cases I manually entered the data.

When you gather tweets from twitter you can save the result as a JSON file. I saved the JSON files in a separate folder with each file being named after the politician. However this was approach was not very good. There are special characters in some of the file names and I had trouble reading the files. Not only that but this approach is not portable since other operating systems have different file name requirements. I got it to work but in the future I think I would have either one big JSON file, or index each politician in both the excel file and the file name.

Getting the Personality Scores

To get the personality percentile scores I used IBM Watson's Personality Insights API (PI). I sent the API the concatenated tweet and it returns a personality scores for that concatenated tweet. You could use any piece of text. It might be fun to get the personality scores for movie characters. The scores include big 5 values, facets, needs, and values. Having these scores I was finally able to analyze the tweets.

Analyzing the Personality Scores

From the start it was decided that we would cluster that data. For this I used R, a vector language used by data scientists. I used the k-means algorithm built in to the base package of R to cluster the data.

After this I analyzed the data by making box plots using Excel. This should've been done before clustering You should have a look at the data before clustering to see attributes that have large variances.

Diet Planner

This project I worked on with a friend for our Database class. We had to develop a database application. We agreed to make a diet planner application with a GUI frontend.

We used MySQL with XAMPP for the database. For the frontend we used a GUI framework for Java, JavaFX. I mostly worked on the database and some parts of the GUI.

There was an interesting problem we came to. We wanted to have a table for each main food category since certain foods have different values. For example only a meat could be labeled as seafood So we wanted the Food table to reference only one of five tables. This was our first database project and we did not realize that such a schema was not possible. We got around this by having a compound key of the food id and the food type.

The GUI was done using code instead of a drag and drop interface. It was a little painful but we managed to get it done. The primary reference videos we were using did not cover GUI development FXML, which is similar to Microsoft's XAML, since they both attempt to separate program logic from the interface.

Garlic

Garlic was a client server application done in a team of two for our Object Oriented Programming class. The goal of the project was to develop an application that emulated the services that the Scholarship and Accounting offices did. The program allowed the user to sign up for scholarships, financial aid, pay their tuition, testing, and parking.

It was done using Java and the sockets class. I worked on the financial aid side of services and the connection and transfer of data between the client and server. An interesting problem I came across was how to share classes. There were initially two packages Client and Server. Each package had to transfer objects of classes that they both implemented. I am not sure why I thought I should make the exact same class files and put them in each package but it did not work since the server did not have access to the Client package's Scholarship object The solution was to have a package of shared classes. However doing this ruined the chances of having separate binaries for the client and server. The solution to that which was not implemented was to have a shared class library (that's the point of DLLs).
